@Library Runtime

@Overview
  Auf Runtime-Objekte (in Deutsch auch "Laufzeit-Objekte") kann durch Verwendung ihrer String-Referenzen
  auch dann zugegriffen werden, wenn das Programm kompiliert wurde.
  @LineBreak
  Für weitere Informationen über das "Runtime"-Konzept siehe @ReferenceLink "Runtime" "Runtime".
  
@CommandList

@ExampleFile All Runtime.pb

@SupportedOS

;--------------------------------------------------------------------------------------------------------

@Function Ergebnis = GetRuntimeInteger(Objekt$)

@Description
  Gibt den Integer-Wert des Runtime-Objekts zurück. Ist das Runtime-Objekt eine Prozedur,
  wird die Prozedur-Adresse zurückgegeben.

@Parameter "Objekt$"
  Name des Objekts, von dem der Wert ermittelt werden soll. Die folgenden Objekte werden
  unterstützt: @LineBreak
- Variable: der Objektname ist 'VariablenName' (ohne Beachtung von Groß- und Kleinschreibung). @LineBreak
- Konstante: der Objektname ist '#KonstantenName' (ohne Beachtung von Groß- und Kleinschreibung). @LineBreak
- Prozedur: der Objektname ist 'ProzedurName()' (ohne Beachtung von Groß- und Kleinschreibung). @LineBreak
  @LineBreak
  Beim Zugriff auf öffentliche @ReferenceLink "module" "Module"-Elemente ist der Modulpräfixname
  obligatorisch, auch wenn @Keyword UseModule verwendet wird.

@ReturnValue
  Gibt den Integer-Wert des angegebenen Objekts zurück oder Null, falls es nicht gefunden wurde.
  Da Null ein gültiger Integer-Wert ist, kann @@IsRuntime verwendet werden, um festzustellen, ob
  das Runtime-Objekt tatsächlich existiert. Ist die Variable vom Typ Float oder Double, wird sie
  automatisch in Integer umgewandelt. Handelt es sich bei dem Runtime-Objekt um eine Prozedur,
  wird die Prozeduradresse zurückgegeben.

@SeeAlso
  @@SetRuntimeInteger, @@IsRuntime

@SupportedOS

;--------------------------------------------------------------------------------------------------------
